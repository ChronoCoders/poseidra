Implementation Roadmap

Five phases to production.

Each phase has concrete completion criteria. A phase is complete when every criterion passes — not before. No skipping, no shortcuts.

Phases

Sequenced. Dependency-aware.

PHASE 01
Circuits + Prover
Agents: circuit-builder · rust-prover-builder
Circuits (circom)
  • commitment.circom — Poseidon(domain_C, secret, nullifier)
  • nullifier.circom — Poseidon(domain_N, nullifier, chain_id, contract)
  • withdraw.circom — full withdrawal circuit
  • Poseidon t=3 implementation
  • Compile, prove, verify scripts
Rust Prover
  • poseidon.rs — matching circuit parameters exactly
  • witness.rs — full witness generation
  • prover.rs — PLONK proof generation
  • domain.rs — domain separation constants
  • Typed errors, zero unwrap() in lib code
Completion Criteria
Constraint count verified: 2,300–2,600
Poseidon output: Rust == circom for 5+ test vectors
Proving time <2s measured on release build
All invalid witness tests rejected
Domain constants documented and shared
cargo build + clippy: zero warnings
PHASE 02
Smart Contracts
Agent: solidity-builder · Prerequisite: Phase 1 complete
Contracts
  • Poseidra.sol — deposit, withdraw, nullifier tracking
  • MerkleTree.sol — Poseidon, depth 20, root history
  • Verifier.sol — generated from circuit via snarkjs
  • ComplianceRegistry.sol — association + exclusion sets
  • Poseidon.sol — matching circuit constants exactly
Tests + Tooling
  • Foundry test suite — 100% coverage
  • Fuzz tests — invalid proof, double spend, stale root
  • Gas snapshot — forge snapshot
  • Deploy script — Deploy.s.sol
  • Local anvil integration test
Completion Criteria
forge build: zero warnings
forge test: all pass, none skipped
Deposit gas <250k (measured)
Withdrawal gas <600k (measured)
Verifier.sol accepts proof from Rust prover
Double spend rejected on-chain
Stale root (101st) rejected on-chain
PHASE 03
Backend + Relayer
Agent: rust-backend-builder · Prerequisite: Phase 2 complete
Backend API
  • Chain indexer — Deposit / Withdrawal events
  • Local Merkle tree state
  • GET /merkle/path/{commitment}
  • GET /roots — last 100 valid roots
  • PostgreSQL: commitments, nullifiers, roots
  • Redis: caching, job queue
Relayer
  • Commit-reveal fee mechanism
  • POST /submit — accept withdrawal + fee commitment
  • GET /status/{job_id}
  • Multiple RPC endpoints, rotation on failure
  • Direct withdrawal fallback
  • Private key: env-only, never logged
Completion Criteria
cargo build + clippy: zero warnings (both crates)
Full deposit → index → path → proof → relay cycle works
Commit-reveal fee cycle tested end-to-end
Direct withdrawal fallback tested
No secret material in any log output
PHASE 04
Compliance Layer
Agent: compliance-builder · Prerequisite: Phase 3 complete
Set Management
  • Association set A — on-chain backed
  • Exclusion set E — sparse Merkle tree
  • Local cache synced from chain events
  • Governance authority validation
  • Timelock enforcement
ZK Proofs + API
  • MembershipProof — C ∈ A
  • NonMembershipProof — C ∉ E
  • Temporal validity — valid_at timestamp
  • Risk scoring API integration + cache
  • POST /compliance/prove/membership
  • POST /compliance/prove/nonmembership
Completion Criteria
Membership proof generates and verifies
Non-membership proof generates and verifies
Temporal validity: proof before flagging valid
Temporal validity: proof after flagging invalid
Governance timelock tested
API key never appears in logs
PHASE 05
Integration + Deploy
Agent: integration-validator · Prerequisite: All phases complete
Integration Tests

  • Cross-client: Rust proof accepted by Solidity verifier
  • Double spend → reverts
  • Stale root (101) → reverts
  • Invalid proof (1 byte corrupt) → reverts
  • MEV ordering simulation
Testnet Deploy

  • Deploy addresses → deployments/{network}.json
  • Gas snapshot on live contracts
  • 48h stability period

Engineering Standards

Non-negotiable across all phases.

0
Zero Warnings
cargo build, cargo clippy, forge build — all clean. No exceptions. No suppressed warnings.
∅
No unwrap()
No unwrap() or expect() outside tests. All errors typed and propagated with context.
⊗
No Secret Leakage
No secret material in logs, API responses, or error messages. Witness computation isolated from network-facing code.
⊕
Measured Gas
Gas figures come from Foundry forge snapshot on deployed contracts. No estimates in production decisions.
