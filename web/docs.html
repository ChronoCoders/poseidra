Tech Stack
Full Stack Reference
Contracts Solidity + Foundry forge build, forge test, forge snapshot for gas measurement
ZK System PLONK + KZG No per-circuit trusted setup. Universal KZG reference string.
Circuits circom → halo2 circom for MVP. Halo2/IPA for L2 recursive aggregation.
Prover Rust Witness generation, Poseidon, PLONK. Zero warnings. No unwrap().
Backend Rust + Axum Chain indexer, Merkle state, REST API. PostgreSQL via sqlx.
Cache / Queue Redis Relayer job queue, API response caching, root history.
Relayer Rust Commit-reveal fee, multi-RPC, direct withdrawal fallback.
Networks zkSync · Scroll · Base · L1 L2-first for gas efficiency. L1 as reference verifier.
Backend API
REST Endpoints

All endpoints return typed JSON. No untyped serde_json::Value in responses.

Merkle State
GET /merkle/path/{commitment}
Returns Merkle path for a commitment — pathElements, pathIndices, and current root. Used by the prover to generate the withdrawal witness.
GET /merkle/root
Returns the current Merkle root.
GET /roots
Returns the list of valid roots — last 100. A withdrawal proof must reference one of these roots.
Commitment Status
GET /commitment/{hash}
Returns commitment status: pending | confirmed | spent.
Relayer
POST /relayer/submit
Submit a withdrawal proof + fee commitment to the relayer queue. Returns job_id. Body: { proof, public_inputs, fee_commitment, fee_amount, nonce }.
GET /relayer/status/{job_id}
Returns job status: pending | submitted | confirmed | failed.
Compliance
GET /compliance/status/{commitment}
Returns compliance status: valid | flagged | unknown.
POST /compliance/prove/membership
Generate a ZK membership proof (C ∈ A) for the given commitment.
POST /compliance/prove/nonmembership
Generate a ZK non-membership proof (C ∉ E) for the given commitment.
Circuit Specification
withdraw.circom
// Private inputs signal input secret; signal input nullifier; signal input pathElements[20]; signal input pathIndices[20]; // Public inputs signal input root; signal input nullifierHash; signal input recipient; signal input chainId; signal input contractAddress; // Constraints // 1. Recompute commitment commitment === Poseidon([DOMAIN_C, secret, nullifier]) // 2. Recompute nullifier hash nullifierHash === Poseidon([DOMAIN_N, nullifier, chainId, contractAddress]) // 3. Verify Merkle inclusion root === MerkleVerifier(commitment, pathElements, pathIndices)

Total constraints: ~2,300–2,600 (Poseidon t=3, depth 20). Verify after every circuit change.

Domain Constants
Cross-Component Consistency

Domain constants must be identical across the circom circuit, Rust prover, and Solidity verifier. Any mismatch is a critical implementation error and will cause all proofs to fail.

// Rust (domain.rs) pub const DOMAIN_COMMITMENT: Fr = Fr::from_raw([...]); pub const DOMAIN_NULLIFIER: Fr = Fr::from_raw([...]); // circom (constants.circom) var DOMAIN_C = ...; var DOMAIN_N = ...; // Solidity (Poseidon.sol) uint256 constant DOMAIN_C = ...; uint256 constant DOMAIN_N = ...; // Rule: DOMAIN_C != DOMAIN_N, both not in {0, 1}

Actual values are set during Phase 1 by the circuit-builder agent and communicated to all other components before implementation begins.

Claude Code Agents
.claude/agents/ Setup

Poseidra uses Claude Code with project-specific sub-agents. Each agent owns its phase and contains concrete implementation tasks, not just review checklists.

Phase 1
circuit-builder.md
Builds all circom circuits. commitment.circom, nullifier.circom, withdraw.circom. Owns domain constant specification. Validates constraint count after every compile.
Phase 1
rust-prover-builder.md
Builds Rust prover crate. poseidon.rs, witness.rs, prover.rs, domain.rs. Cross-verifies Poseidon output with circuit before proceeding to proof generation.
Phase 2
solidity-builder.md
Builds all Solidity contracts. Generates Verifier.sol from final circuit. Measures gas via Foundry snapshot. Checks-effects-interactions pattern enforced.
Phase 3
rust-backend-builder.md
Builds backend API and relayer. Indexer, Merkle state, REST API, commit-reveal fee mechanism, anti-censorship fallback. Private key never logged.
Phase 4
compliance-builder.md
Builds compliance layer. Association and exclusion set ZK proofs, temporal validity enforcement, risk scoring API integration with caching.
Phase 5
integration-validator.md
Validates full system. End-to-end deposit-to-withdrawal cycle, cross-client consistency, MEV simulation, testnet deploy, pre-mainnet checklist.
Every session
session-documenter.md
Appends to docs/session-log.md. Records completed work, decisions, constraint counts, gas numbers, open issues, and concrete next steps. Mandatory at session end.
Engineering Standards
Non-Negotiables
  • Zero warnings — cargo build, cargo clippy, forge build all clean
  • No unwrap() or expect() outside test code
  • All errors typed, propagated with context — no anyhow in library code
  • No secret material in logs, API responses, or error messages
  • Gas numbers measured via forge snapshot — never estimated in production decisions
  • Constraint count verified after every circuit change
  • Domain constants verified identical across all components before any proof test
  • Git author: ChronoCoders / altug@bytus.io only — no co-authors, no Claude attribution
  • Groth16 never used — PLONK/KZG only
  • Flagging authority never hardcoded — governance parameter only
  • Nullifier always includes chain_id and contract_address
  • Root validity window always enforced — stale proofs always rejected
